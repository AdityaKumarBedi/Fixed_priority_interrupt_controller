# Fixed_priority_interrupt_controller
This Verilog FPIC latches 4 IRQs to prevent missed events. A nested encoder prioritizes IRQ3 down to IRQ0. It handles CPU comms via a valid/ack handshake, clearing only the active interrupt using a shifting bitmask while keeping other pending requests alive.
4-Channel Fixed-Priority Interrupt Controller (FPIC)This project implements a 4-channel Fixed-Priority Interrupt Controller in Verilog HDL. It serves as a smart hardware bridge between peripheral devices and a CPU, offloading event management so the processor doesn't waste clock cycles on inefficient polling.How It WorksThe controller manages peripheral requests and coordinates with the CPU using three main steps. First, it catches the request by instantly saving peripheral interrupts into an internal register. This ensures that even short, transient pulses are caught and held until the CPU is ready. Next, it handles priority sorting. If multiple devices request attention at the exact same time, a nested priority encoder settles the conflict by following a strict hardwired hierarchy where IRQ3 has the highest priority and IRQ0 has the lowest.Finally, it coordinates the CPU handshake. When an interrupt is ready, the controller alerts the CPU by pulling the valid line high and placing the active channel's binary ID on the interrupt ID bus. Once the CPU handles the event, it sends back an acknowledgment pulse. The controller then uses a shifting bitmask to clear only that specific handled interrupt from its memory, leaving any other pending requests completely untouched and ready for the next cycle.Interface SignalsThe module connects to the system using a straightforward interface. For inputs, it takes a system clock signal and an active-high reset line to clear all internal registers. The peripheral devices connect through a 4-bit wide raw interrupt request bus, while the CPU communicates its completion status via a single-bit acknowledgment pulse input. For outputs, the controller features a single-bit valid signal to alert the CPU when an interrupt needs attention, alongside a 2-bit ID bus that tells the CPU exactly which channel to service.
